# タスク管理アプリ 要件定義書

## 1. アプリ概要

| 項目 | 内容 |
|------|------|
| アプリ名 | タスク管理アプリ（Trello風） |
| 目的 | 自分のタスクを「未着手・進行中・完了」の3段階で視覚的に管理する |
| 対象ユーザー | 個人でタスクを管理したい人 |

---

## 2. 画面構成

| 画面 | 説明 |
|------|------|
| ボード画面（メイン） | タスクカードを3列（未着手・進行中・完了）に並べて表示するメイン画面 |
| タスク作成モーダル | タスクの新規作成・編集に使うポップアップウィンドウ |

---

## 3. タスクのデータ項目

各タスクが持つ情報です。

| 項目名 | 説明 | 必須 |
|--------|------|------|
| タイトル | タスクの名前（例：「資料作成」） | 必須 |
| 説明文 | タスクの詳細メモ | 任意 |
| 優先度 | 高・中・低の3段階 | 必須 |
| 期限 | 完了させたい日付 | 任意 |
| ステータス | 未着手・進行中・完了（カラムの位置と連動） | 自動 |

---

## 4. 機能一覧

### 基本操作（CRUD）

| 機能 | 説明 |
|------|------|
| タスク作成 | ボタンを押してタスクを新規追加する |
| タスク表示 | タスクをカード形式で一覧表示する |
| タスク編集 | カードをクリックしてタイトル・説明文・優先度・期限を変更する |
| タスク削除 | カード上の削除ボタンでタスクを消す |

### ステータス管理

| 機能 | 説明 |
|------|------|
| 3列カンバンボード | 「未着手」「進行中」「完了」の3つのカラムを表示する |
| カードのドラッグ＆ドロップ | カードをつかんで別のカラムに移動できる |

### データ保存

| 機能 | 説明 |
|------|------|
| 自動保存 | タスクの追加・編集・削除のたびにブラウザのlocalStorageへ自動保存する |
| データ永続化 | ブラウザを閉じても次回開いたときにタスクが残っている |

---

## 5. 技術スタック（使用技術）

| 役割 | 技術 | 理由 |
|------|------|------|
| 見た目・構造 | HTML / CSS | 基本の見た目を作る |
| 動き・操作 | JavaScript | ドラッグ＆ドロップや保存の処理を担当 |
| データ保存 | localStorage（ブラウザ内蔵） | サーバーなしでブラウザだけでデータを保存できる |

> **localStorageとは？**  
> ブラウザが持っている小さなメモ帳のような仕組みです。  
> サーバーやデータベースを用意しなくても、パソコンのブラウザの中にデータを保存できます。  
> ブラウザを閉じてもデータは消えません。（ブラウザのキャッシュを消すと消えます）

---

## 6. 今回は作らないもの（対象外）

- ログイン・ユーザー登録機能
- 複数人での共有・共同編集
- サーバー・データベースを使ったデータ保存
- スマートフォン向けの最適化
